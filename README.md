# Teeklit – Community Module

**은둔·고립 청년 회복 지원형 실천 커뮤니티 앱**

본 README는 Teeklit 프로젝트 중 **커뮤니티 기능 개발에 대한 개인 기여 내용**을 중심으로 정리한 문서입니다.

---

## Project Overview

Teeklit은 은둔·고립 청년이 **작고 실행 가능한 행동(‘티클’)**을 통해

일상 리듬과 자기효능감을 회복할 수 있도록 돕는 실천 중심 커뮤니티 앱입니다.

본 프로젝트에서 저는 **커뮤니티 UI 및 핵심 기능 전반을 담당**하였습니다.

- **플랫폼:** Android / iOS
- **개발 형태:** 팀 프로젝트
- **개발 기간:** 2025.09 – 2025.12
- **담당 영역:** 커뮤니티 기능 설계 및 구현

---

## My Role – Community Developer

### 담당 범위 요약

- 커뮤니티 화면 UI 구현
- 게시글·댓글·대댓글 CRUD 기능 개발
- 커뮤니티 상호작용 로직(좋아요, 신고, 차단) 구현
- Firestore 기반 커뮤니티 데이터 구조 설계
- 익명 기반 커뮤니티 UX 설계

## Implemented Features

### 1. 커뮤니티 메인 화면

<img width="1284" height="2778" alt="KakaoTalk_20251229_120839703" src="https://github.com/user-attachments/assets/01157705-14c6-4138-ab22-39a45c7289f9" />

- 게시판 카테고리 토글 (인기 / 자유 등)
- 게시글 목록에서 제목, 내용 미리보기, 카테고리, 댓글 수 표시
- 인기 게시글 노출을 통한 간접 참여 유도

**기여 포인트**

- 읽기만 해도 참여감을 느낄 수 있도록 정보 밀도 조절
- 초기 진입 시 글 작성 부담을 줄이는 UX 구성

---

### 2. 게시글 CRUD

- 게시글 작성 / 수정 / 삭제 기능 구현
- 카테고리 선택 및 이미지 업로드 지원
- Firestore + Firebase Storage 연동

**기여 포인트**

- 커뮤니티 성격에 맞는 최소 입력 UX 설계

- 이미지 포함 게시글에서도 데이터 정합성 유지

---

- 부적절한 게시글 대응을 위한 관리 흐름 고려

**기여 포인트**

## UX & Design Considerations

- **익명 기반 참여**
    - 닉네임만 사용
    - 개인 정보 노출 최소화
- **간접 참여 허용**
    - 글을 작성하지 않아도 읽기만으로 사회적 연결 경험 제공
- **익숙한 게시판 UX**
    - 학습 비용을 최소화하기 위해 일반적인 게시판 패턴 채택

---

## Tech Stack (Community)

- **Frontend:** Flutter
- **Routing:** go_router
- **State Management:** (프로젝트 내 사용 방식 기준)
- **Database:** Cloud Firestore (NoSQL)
- **Storage:** Firebase Storage
- **Auth 연계:** Firebase Authentication (유저 식별)

---

## Data Structure (Community)

- **Post**
    - 작성자 UID
    - 카테고리
    - 내용
    - 이미지 URL
    - 좋아요 수 / 댓글 수
    - 생성·수정 시간
- **Comment**
    - 게시글 ID 참조
    - 부모 댓글 ID (대댓글)
    - 작성자 UID
    - 내용
    - 생성 시간

Firestore의 **유연한 스키마 구조**를 활용하여

댓글·대댓글 확장에 유리하도록 설계했습니다.

## Problem & Solution

### 문제

- 은둔·고립 청년 대상 서비스 특성상
    
    **글 작성 자체가 심리적 부담이 될 수 있음**
    

### 해결

- 익명 기반 구조
- 읽기만 해도 참여가 가능한 커뮤니티 설계
- 좋아요·댓글 같은 가벼운 상호작용 제공
